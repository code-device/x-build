# 快速上手

本节将介绍如何在全局环境安装 X-BUILD 和通过命令行快速的生成项目。

# 环境搭建

  • 需要在本地安装 Node.jsopen in new window 和 Gitopen in new window。
  • 推荐使用 vscodeopen in new window 作为代码编辑器，安装以下插件可以更好的体验脚手架：
    • Vue Language Features (Volar)open in new window（Vue3 开发必备）
    • ESLintopen in new window（Javascript 代码检查）
    • Stylelintopen in new window（CSS 代码检查）
    • Prettier - Code formatteropen in new window（代码格式化）
    • Tailwind CSS IntelliSenseopen in new window（Tailwind 提示插件）

# 安装 CLI

我们推荐使用 npm 的方式进行全局安装：

npm install x-build@next -g

yarn global add x-build@next

mac、linux 用户需要权限增加 sudo。

使用方式类似于 @vue/cli，提供了终端里的 x-build 或 x（简写）命令。

# 创建项目

# 命令

通过以下指令创建一个新项目：

x-build create [name]

x create [name]

# 选择特性

使用创建命令会提示一系列问题，需要你根据实际需求做出选择：

# 选择默认 CSS 预处理器

  • Scss
  • Less

# 选择默认组件库

  • Element Plus
  • Ant Design Vue
  • Naive UI
  • Vant

# 安装依赖

  • 输入命令后，模板文件会自动创建名字为[name] 的文件夹，并将模板文件复制到此文件夹内。
  • 自动初始化 Git。
  • 依赖会自动安装，会跟你你所安装的包管理工具按优先级选择：

优先级：yarn > npm

提示

强烈建议不要用直接使用 cnpm 安装，会有各种诡异的 bug，可以通过重新指定 registry 来解决 npm 安装速度慢的问题。若还是不行，可使用 yarn 替代 npm。

# 开发环境

根据安装后的相关提示，输入相关命令以继续：

npm run dev

npm run analyzer

npm run lint

npm run stylelint

提示

通常情况下 ESLint 与 Stylelint 没必要单独执行，husky 提供的 git commit 钩子会在你提交代码时对你缓存区里的代码做校验，这会节省你很多的时间。
